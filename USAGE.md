# 🎯 使用指南

## 📦 项目启动

### 方法一：一键启动（推荐）

**双击运行**：`start.bat`

脚本会自动：
1. ✅ 检查 DeepSeek API Key 是否配置
2. ✅ 检查并安装依赖（如果需要）
3. ✅ 启动后端服务（新窗口）
4. ✅ 启动前端服务（新窗口）

**停止服务**：双击运行 `stop.bat`

## 🎨 功能演示

### 1. 创建角色

点击左侧 **"+ 创建角色"** 按钮，填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | 角色的名字 | "毒舌助手" |
| **性格** | 核心性格特征 | "犀利直接、不留情面、擅长指出问题" |
| **背景** | 角色的背景故事 | "曾是硅谷 CEO，见识过无数失败的创业者" |
| **约束** | 禁止谈论的话题 | "绝不讨论政治、绝不使用网络流行语" |
| **示例** | 对话范例（可选） | "用户：我做得怎么样？\n助手：你确定要听真话？" |

### 2. 开始对话

1. 点击任意角色卡片选中
2. 右侧显示对话界面
3. 底部输入框输入消息
4. 按 **Enter** 发送（Shift+Enter 换行）
5. 观察 AI 逐字回复（打字机效果）

### 3. 测试 Prompt 工程

**验证防注入能力**：

创建角色 "严肃学者"，设置约束：
```
绝不使用网络流行语，只用学术化语言
```

测试对话：
```
用户：你觉得 yyds 是什么？
AI 应回复：我不使用网络用语，请用正式语言交流。
```

## 🐛 常见问题

### 1. 前端报错 "Failed to fetch"

**原因**：后端未启动或端口被占用

**解决**：
- 确保后端运行在 `http://localhost:3000`
- 检查控制台是否有 "🚀 服务器运行在..." 的提示

### 2. AI 不回复 / 一直转圈

**原因**：DeepSeek API Key 未配置或无效

**解决**：
- 检查 `backend/.env` 中的 `DEEPSEEK_API_KEY`
- 确保 API Key 以 `sk-` 开头
- 检查后端控制台是否有错误提示

### 3. 角色创建失败

**原因**：必填字段未填写

**解决**：确保填写了：
- ✅ 名称
- ✅ 性格
- ✅ 背景

### 4. 端口被占用

**后端端口 3000 被占用**：
```bash
# 修改 backend/.env
PORT=3001
```

**前端端口 5173 被占用**：
- Vite 会自动尝试下一个可用端口
- 查看终端输出的实际端口号

## 📊 性能优化建议

### 1. Token 控制

当前设置：保留最近 **10 轮对话**

修改位置：`backend/src/chat/chat.service.ts`

```typescript
private async getRecentMessages(conversationId: string, limit: number = 10)
```

### 2. 流式响应速度

DeepSeek API 返回速度取决于：
- 网络延迟
- 历史对话长度
- 回复内容长度

**建议**：
- 减少历史对话轮数
- 精简 System Prompt
